# brAIn-map テストガイド

## 概要

このプロジェクトには、性質の異なる 2 種類のテストスイートが含まれています。

1.  **AI ストーリーテスト (`npm run story`)**: AI（Claude）を使用したエンドツーエンドの半自動テスト。非決定的な動作を人間が評価します。
2.  **MCP 統合チェック (`npm run test:mcp`)**: AI を使用しない決定的な自動テスト。MCP サーバーのツールが正しく動作するかをプログラムで検証します。

---

## セットアップ（初回のみ）

```bash
npm run build
```

---

## 1. AI ストーリーテスト (Semi-Automated)

AI の出力は非決定的なので「正解と完全一致」での自動テストは難しい。このテストスイートは以下を組み合わせた**半自動テスト**を採用している：

| 自動チェック | 人間が評価 |
|---|---|
| 構造的整合性（エッジが存在するノードを参照しているか等） | 内容の質・論理的適切さ |
| ツール呼び出し統計 + 効率性警告 | グループ名・エッジラベルの妥当性 |
| ノード数・エッジ数・グループ数の assertions | ブラウザでの視覚的確認 |

### 実行

```bash
npm run story 01    # または 02 03 04 05
```

`BRAIN_MAP_PORT` で使用ポートを変更できる（デフォルト: 3001）。

---

## 2. MCP 統合チェック (Deterministic)

AI を介さず、MCP クライアントから直接ツールを呼び出して動作を検証します。リグレッション防止のための「冪等性のあるテスト」です。

### 実行

```bash
npm run test:mcp
```

## ファイル構成

```
tests/
├── stories/                   # AI ストーリーテスト関連
│   ├── runner.ts              # ストーリー実行 CLI
│   ├── data/                  # ストーリー定義ファイル群
│   └── results.log            # 実行結果の記録
├── mcp/                       # MCP 統合チェック関連
│   └── runner.ts              # MCP チェック実行 CLI
├── lib/                       # テスト用共通ライブラリ
│   ├── api.ts                 # REST API クライアント
│   ├── mcp-client.ts          # MCP SSE クライアント
│   ├── server.ts              # サーバープロセス管理
│   ├── display.ts             # ターミナル出力装飾
│   ├── checks.ts              # 構造整合性チェック
│   └── types.ts               # 型定義
├── .gitignore                 # テスト用一時ファイルの除外設定
└── README.md                  # このファイル
```
